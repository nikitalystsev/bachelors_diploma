# Модуль модификации поведения робота Ф-2

Программная часть дипломного проекта: графическое приложение для получения активированных д-сценариев от робота Ф-2 и их локального перевзвешивания с учетом профиля робота, отношения к собеседнику и параметров текущей ситуации.

Проект реализован на Python. Графический интерфейс построен на Tkinter, клиент Ф-2 использует HTTP-запросы через `requests`, фаззификация численных признаков выполняется через `simpful`, а служебный загрузчик GIF использует `beautifulsoup4`.

## Возможности

- отправка реплики пользователя в API Ф-2 и извлечение пар `scenario`/`proximity` из JSON-ответа;
- выбор профиля робота и настройка контекста в GUI;
- фаззификация численных признаков с треугольными и трапециевидными функциями принадлежности;
- применение JSON-правил к активированным д-сценариям;
- расчет итогового веса сценария с учетом множителей эффектов;
- отображение исходного веса, итогового множителя, нового веса и сработавших правил;
- функциональные тесты клиента Ф-2, загрузчика конфигураций и фильтра сценариев.

## Структура проекта

```text
src/
  main.py                         # точка входа в GUI-приложение
  f2robot_client.py                # HTTP-клиент API Ф-2
  f2robot_profile_config.py        # загрузка JSON-конфигураций профилей
  f2robot_personality_filter.py    # фаззификация, правила и перевзвешивание сценариев
  gui/
    GUI.py                         # графический интерфейс
    config_GUI.py                  # параметры оформления GUI
  profiles/
    abstract/                      # абстрактный профиль-конструктор и общие правила
    authoritarian/                 # профиль "Авторитарно-контролирующий"
    distanced/                     # профиль "Защитно-дистанцированный"
  data/
    f2robot_gifs/                  # локальные GIF-анимации жестов Ф-2
    rule_effects.json              # множители и подписи эффектов правил
    trait_labels.json              # русскоязычные подписи черт профиля
tests/                             # функциональные тесты
```

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

На Windows используйте активацию окружения из `.venv\Scripts\activate`.

## Настройка API Ф-2

Для запуска GUI нужен адрес API Ф-2. Его можно задать переменной окружения:

```bash
export F2_ROBOT_API_LINK="http://host:port/path"
```

Или добавить в файл `.env` в корне проекта:

```text
F2_ROBOT_API_LINK=http://host:port/path
```

Адрес нужно указывать полностью, включая схему `http://` или `https://`: клиент использует значение без автоматического добавления схемы. Если переменная задана и в окружении, и в `.env`, значение из `.env` перезаписывает окружение. Тайм-аут запроса к API составляет 30 секунд.

## Конфигурация профилей и правил

Профили робота находятся в `src/profiles/<profile_id>`. Если профиль явно не задан, загрузчик выбирает первый конкретный профиль в лексикографическом порядке и пропускает профиль `abstract`, чтобы приложение не открывалось в режиме конструктора без явного выбора пользователя.

Каждый профиль задается набором JSON-файлов:

- `trait_profile.json` — идентификатор профиля, отображаемое название и значения персональных черт робота;
- `relation_params.json` — параметры отношения к собеседнику;
- `situation_params.json` — параметры текущей ситуации, включая fuzzy-термы для численных шкал;
- `rules.json` — условия правил, целевые д-сценарии и эффекты перевзвешивания.
- `robot_gif.json` — путь к GIF-анимации робота для конкретного профиля.

Профиль `abstract` является специальным. Он не содержит `robot_gif.json` и выполняет две роли:

- при прямом выборе работает как конструктор персонажа: все 30 граней личности отображаются выпадающими списками с термами `low`, `medium`, `high`;
- при выборе любого конкретного профиля добавляет к нему свои правила, параметры отношения и параметры ситуации, не заменяя правила и параметры выбранного профиля.

Для конкретных профилей черта в `trait_profile.json` задается фиксированным значением:

```json
{
  "assertiveness": {
    "value": "high",
    "label": "высокая"
  }
}
```

Для профиля `abstract` черта задается набором доступных термов:

```json
{
  "assertiveness": {
    "terms": [
      {"term": "low", "label": "низкая"},
      {"term": "medium", "label": "средняя"},
      {"term": "high", "label": "высокая"}
    ]
  }
}
```

Формат `robot_gif.json`:

```json
{
  "gif_path": "src/data/f2robot_gifs/introversion6.gif"
}
```

Относительный путь в `gif_path` задается от корня проекта. Абсолютный путь тоже поддерживается.
Если указано только имя файла, например `"appeal6.gif"`, оно ищется в `src/data/f2robot_gifs`.

Общие справочники лежат в `src/data`:

- `rule_effects.json` — множители `strong_decrease`, `decrease`, `no_change`, `increase`, `strong_increase` и их подписи;
- `trait_labels.json` — подписи черт профиля для интерфейса.

Чтобы добавить новый конкретный профиль, создайте каталог `src/profiles/<profile_id>` и положите в него профиль, параметры отношений, параметры ситуации, правила и `robot_gif.json`. Файлы с общими справочниками можно не копировать: загрузчик сначала ищет JSON в каталоге профиля, затем в `src/data`.

Если в `src/profiles/abstract` заданы общие `rules.json`, `relation_params.json` или `situation_params.json`, они автоматически добавляются к любому конкретному профилю. При прямом выборе `abstract` используются только файлы из `src/profiles/abstract`; правила и параметры конкретных профилей в этом режиме не подключаются.

Текущий валидатор конфигурации строгий: все термы, объявленные в `trait_profile.json`, `relation_params.json` и `situation_params.json`, должны встречаться хотя бы в одном условии правил. Если какой-то терм не покрыт правилами, загрузка профиля завершится ошибкой `Rules do not cover configured terms`.

## Формат параметров

Для дискретного выбора используйте `input: "select"` и список `options`:

```json
{
  "feature": "social_circle",
  "label": "Круг общения",
  "input": "select",
  "default": "distant",
  "options": [
    {"term": "close", "label": "близкий"},
    {"term": "distant", "label": "дальний"}
  ]
}
```

Для численного признака используйте `input: "slider"`, границы шкалы и `fuzzy_terms`:

```json
{
  "feature": "offensiveness_degree",
  "label": "Степень обидности",
  "input": "slider",
  "default": 0.78,
  "minimum": 0.0,
  "maximum": 1.0,
  "fuzzy_terms": [
    {
      "term": "high",
      "label": "высокая",
      "function_type": "trapezoid",
      "points": [0.55, 0.75, 1.0, 1.0]
    }
  ]
}
```

Поддерживаются `function_type: "triangle"` с тремя точками и `function_type: "trapezoid"` с четырьмя точками. Численные значения ограничиваются заданными границами шкалы.

## Формат правила

```json
{
  "rule_id": "R05_medium_offensiveness_raises_sadness",
  "label": "средняя степень обидности усиливает сценарий грусти",
  "conditions": [
    {
      "feature": "offensiveness_degree",
      "terms": ["medium"]
    }
  ],
  "target_scenarios": ["ты-грустишь"],
  "effect": "increase"
}
```

Правило срабатывает для сценариев из `target_scenarios`, если условия по признакам профиля, отношения и ситуации совпали с заданными термами. Для fuzzy-признаков сила срабатывания зависит от степени принадлежности терму.

Для каждого `effect` из правил должен быть задан численный множитель и подпись в `rule_effects.json`.
